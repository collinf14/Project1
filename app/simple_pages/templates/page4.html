CI/CD

Continuous Integration and Continuous Deployment is the practice of running a program fully while editing the source code. CI/CD allows for testing and experimentation as a developer is going through the process of a building a project. It is also used to get apps to consumers as quickly as possible and allow for changes based on the consumers wants and needs as it is seen fit.

Project Setup

March 7, 2022 by Collin Fogerty

This project was set up and continuous integrated and deployed primarily through GitHub, DockerHub, and Heroku. Through GitHub, this project ran on a main base branch and then had plenty of other branches to push changes to when the contents of the source code was updated. The method of PyTest through a workflows was triggered to check before merging a pull request. This allowed for errors to be noticed and corrections to be made before truly affecting the main branch. Deploying master to a production server came when the pull requests were fully authorized and the branches were merged into the main branch. Time after time the side branches were checked and pulled together into the main branch in order to create the final product.

When the main branch was actually updated, this triggered a deployment to Dockerhub so that the container image was also updated. This was also done through one of the workflows check as it allowed for connection to Docker Hub in the source code.
