Object Oriented Programming in Calculator

The Calculator is coding in the style of Object Oriented Programming. The properties of OOP are the terms encapsulation, inheritance, polymorphism, and abstraction. These terms define OOP and without these core concepts involved a project cannot be considered object oriented. Their interpretations within the calculator an OOP project are described below.

Demonstration of Principles

March 28, 2022 by Collin Fogerty



Encapsulation

In this calculator, there are a couple instances of where a class keeps its method or properties private. For the most part, this is because those functions are necessary for that class and really do not have any use in other classes. In the static methods, mutli and div, below they both have a variable called calc that is used for calculations. But each variable is encapsulated only to their respective methods. Therefore, even though they have the same name they do not share value and instead stay private within the methods. This helps because they are essentially doing the same thing so it makes it much easier to name them the same so that another programmer would understand their similarity but they each work independently and do not affect each other so they both can work properly.

Example of Encapsulation in the Calculator

Inheritance

In this calculator, the gaining of methods and properties is essential to how the calculations of addition, subtraction, multiplication, and division work. As evidenced in the example below, the addition class needs to inherit the conversion of a tuple list to deal with the possibility of floating point numbers. Without the child class of Addition inheriting the method from its parent, Calculation, it would not be able to deal with floating point numbers and therefore not being a very accurate calculator.

Example of Inheritance in the Calculator

Polymorphism

The 'many shapes' property of OOP in this calculator means that objects in the classes of operations need to be able to do some of the same things, such as get result, but each in their own unique way. Addition needs to be able to add to get the result, Subtraction needs to subtract numbers to get the result, so on and so forth. This get_result() method needs to be able to take many shapes when it is used in the different objects of classes.

Example of Polymorphism in the Calculator

Abstraction

Abstraction mostly means keep some things private and others a bit more public. In most cases the private matters are implementation of a mechanism and public maker are the high level function that allow other to use the implementation. In the example for the calculation below, the Calculator class allows all others to see the add method and use that method to add numbers. But all the implementation with the calculations creating and getting the result stay private as other classes do not need that information they just need the addition function to work.
