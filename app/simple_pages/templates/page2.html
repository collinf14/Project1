Containers, Images, and Docker Commands

March 7, 2022 by Collin Fogerty

Containers are the executable versions of source code files. Containers allow for developers to output their projects to run on any infrastructure. Containerization and virtualization are closely related yet containers are more portable and efficient. This is because all containers run on the same operating system where as virtual machines all use individual operating systems.

A container image is like a snapshot of executable code that can be run on any operating system. The source code files of a project build the container image so that it can run as intended by the developer. These images are designed to be the quickest way to get an application running and tested on. Container images on a local machine can allow a user to make changes and see their application develop in a step by step process.



Docker Commands

docker build
Builds a container image. How to start a project on a local machine.
docker run
Starts a container for a project.
docker -version
Shows what version of docker is being used.
docker ps
Shows a list of the containers running with their container IDs, ports, and status.
docker stop
Stops a running container.
docker commit
Creates a new image of a container.
docker push
Pushed a container image onto a repository.
docker pull
Pulls a container image from a repository.
docker rm
Removes a container.
docker login/logout
Used to login and out of a Docker Hub account.
